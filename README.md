# World Clock Multi-Board v4.7

This package preserves the multi-board clock behavior while adding the v4.7
map-library and coordinate-entry features. It retains:

- all update schedules;
- captive-portal behavior;
- runtime web configuration;
- diagnostics and map maintenance;
- SD map caching;
- Sun, Moon, ISS, and orbit-track behavior;
- `CONFIG_BUTTON_HOLD_MS`;
- application state and redraw flow.

## v4.7 map library and coordinate-entry update

Daylight images now live in `/maps` and are discovered at runtime. Each image
gets its own same-name `.rgb565` cache, while `/earth_night.png` and
`/earth_night.rgb565` remain shared. The Maps page provides original-PNG
previews, validation/cache state, selection, and cache rebuild actions. The
selected filename is persistent and automatically falls back if its PNG is
removed.

The setup portal and normal Settings page now use North/South and East/West
selectors plus unsigned numeric magnitude fields for mobile-friendly coordinate
entry.

Recommended SD-card contents:

```text
/maps/earth_day.png
/maps/earth_day_political.png
/earth_night.png
```

The `.rgb565` files are generated by the firmware. Existing installations with
a root `/earth_day.png` are migrated into `/maps/earth_day.png` automatically.


## Select a board

Change one line in `config.h`:

```cpp
#define WORLDCLOCK_BOARD BOARD_HELTEC_WROOM_28
```

Supported selections:

```cpp
BOARD_HELTEC_WROOM_28
BOARD_E32R28T
BOARD_ELEGOO_EL_EB_009
```

## Board profiles

All board-specific values are in `board_profiles.h`:

- TFT pins;
- SD pins;
- panel native geometry;
- display rotation;
- RGB/BGR order;
- display inversion;
- backlight pin and active level;
- configuration-button pin;
- nominal flash size.

The application continues to use its existing constant names through aliases in
`config.h`. This minimizes the code diff and avoids changing behavior.

## Arduino IDE

Use:

```text
Board: ESP32 Dev Module
CPU Frequency: 240 MHz
```

Set Flash Size to match the physical board:

- Heltec WROOM profile: normally 8 MB
- E32R28T: normally 4 MB
- Elegoo EL-EB-009 CYD: normally 4 MB

Use a partition scheme with enough application space for the compiled sketch.

## Profile notes

### Heltec WROOM

The default profile exactly reproduces the working v3.4 settings:

- rotation 3;
- standard 240×320 ILI9341 geometry;
- `rgb_order = false`.

### E32R28T

Preserves the proven unusual settings:

- rotation 4;
- 320×240 panel declaration;
- `rgb_order = true`.

### Elegoo EL-EB-009 CYD

Starts with standard CYD settings:

- rotation 1;
- standard 240×320 geometry;
- `rgb_order = false`.

If the Elegoo display is landscape but upside down, change its rotation in
`board_profiles.h` from `1` to `3`. If only red and blue are exchanged, change
its `rgbOrder`.

## Map image requirements

Every source image must be a non-interlaced 320 × 240 Plate Carrée world map
with Greenwich centered, the Date Line at the edges, and the poles at the top
and bottom. See `Map requirements.txt` for the full checklist.
